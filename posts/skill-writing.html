OpenClaw Skill 开发指南

从 SPA 数据提取专家 skill 的开发过程中总结的实战经验

引言

OpenClaw 是一个强大的 AI Agent 平台，通过 Skills 机制可以扩展其能力边界。 最近我开发了一个 spa-extractor skill，专门用于从 JavaScript 动态渲染的单页面应用中提取数据。 这篇文章将分享开发过程中的核心经验和"死磕"技巧。

什么是 OpenClaw Skill？

Skill 是 OpenClaw 的模块化扩展单元，本质上是一个包含以下内容的文件夹：

  • SKILL.md - 技能的核心文档，包含使用说明和最佳实践
  • scripts/ - 可执行脚本（Python、Bash 等）
  • references/ - 参考资料和文档
  • assets/ - 模板、图片等资源文件

当用户提出与 skill 描述匹配的需求时，OpenClaw 会自动加载该 skill， 按照 SKILL.md 中的指导执行任务。

核心设计原则

1. 渐进式披露 (Progressive Disclosure)

Skill 采用三级加载系统来管理上下文：

  1. 元数据（始终加载）- name + description，约 100 词
  2. SKILL.md 正文（触发后加载）- 详细说明，<5000 词
  3. 资源文件（按需加载）- scripts、references 等
设计要点：description 字段是 skill 的主要触发机制， 必须清晰描述技能的用途和触发条件。不要把"何时使用"放在正文里， 因为正文是在触发后才加载的！

2. 简洁为王

上下文窗口是公共资源。Skill 需要与系统提示、对话历史、其他 skills 共享上下文。

  • SKILL.md 正文控制在 500 行以内
  • 优先使用代码示例而非冗长解释
  • 将详细信息移到 references/ 文件

实战：SPA 数据提取 Skill

让我们深入分析 spa-extractor skill 的设计。这个 skill 解决的核心问题是： 现代 SPA（Vue/React/Angular）初始 HTML 只是一个空壳，所有内容都是异步加载的。

核心挑战

典型的 SPA HTML 结构

<!DOCTYPE html>
<html>
<head>...</head>
<body>
    <!-- 初始只有这个空 div -->
    <div id="app"></div>
    
    <script src="/app.js"></script>
    <!-- JS 执行后才会渲染内容 -->
</body>
</html>

三大死磕戒律

戒律 1：禁止秒读

网页加载或点击"查询"后，绝对不允许立刻截取 DOM！ 必须等待 Vue/React 完成 DOM 重绘。

❌ 错误做法

openclaw browser click "查询"
openclaw browser snapshot  # ← 数据还没渲染！

✅ 正确做法

openclaw browser click "查询"
sleep 5  # 至少等待 3-5 秒
openclaw browser snapshot  # ← 现在数据存在了

戒律 2：文本定位优于类名

Webpack/Vite 等打包工具会混淆 CSS 类名，依赖 .table-row_a1b2c 这样的选择器是极其脆弱的。

❌ 脆弱的选择器

openclaw browser click ".btn-primary_a3f8d"

✅ 健壮的做法

# 使用 JavaScript 文本匹配
openclaw browser evaluate "
    Array.from(document.querySelectorAll('button'))
        .find(el => el.textContent.includes('查询'))
        .click()
"

戒律 3：强制降级策略

如果 openclaw browser click 失败两次，立即降级使用 JavaScript 执行点击。

降级模式

# 尝试 1：视觉点击
openclaw browser click "查询" || FAILED=1

# 尝试 2：再次视觉点击
openclaw browser click "查询" || FAILED=2

# 尝试 3：JS 执行（降级）
if [ $FAILED -eq 2 ]; then
    openclaw browser evaluate "
        Array.from(document.querySelectorAll('button'))
            .find(el => el.textContent.includes('查询'))
            .click()
    "
fi

四步提取 SOP

Step 1: Hydration & 条件穿透

确认 <div id="app"> 已经生成子节点。如果是空的，原地休眠重试。 

# 检查页面是否已渲染
openclaw browser evaluate "
    document.getElementById('app').children.length > 0
"

Step 2: 查询与动态等待

点击查询后，不要相信 Network 面板。数据可能通过 WebSocket 或复杂封装的 XHR 返回。 必须盯着页面，直到 loading 状态消失。 

# 循环检查 loading 状态
for i in {1..10}; do
    is_loaded=$(openclaw browser evaluate "
        !document.querySelector('.el-loading-mask') && 
        !document.querySelector('.loading') &&
        document.querySelectorAll('table tr').length > 1
    ")
    if [ "$is_loaded" = "true" ]; then break; fi
    sleep 2
done

Step 3: 暴力扒取数据

当 DOM 嵌套过深（几十层 div），不要手动解析。直接用 JavaScript 展平： 

openclaw browser evaluate "
    // 策略 1：表格结构
    let rows = Array.from(document.querySelectorAll('table tr, .el-table__row'));
    
    // 策略 2：列表/网格结构（备选）
    if (rows.length === 0) {
        rows = Array.from(document.querySelectorAll('.list-item, .grid-item'));
    }
    
    // 提取为二维数组
    return rows.map(row => {
        const cells = row.querySelectorAll('td, th, .cell');
        return Array.from(cells).map(c => c.innerText.trim()).filter(Boolean);
    }).filter(row => row.length > 0);
"
数据校验：如果抓取结果是 ['-'] 或 ['暂无数据']， 说明等待时间不够，返回 Step 2 重新执行！

Step 4: 数据落表

使用 Python pandas 进行最终校验和 Excel 持久化。

save_to_excel.py 核心逻辑

import pandas as pd
import json

# 加载提取的数据
with open('extracted_data.json') as f:
    data = json.load(f)

# 转换为 DataFrame
headers = data[0]
rows = data[1:]
df = pd.DataFrame(rows, columns=headers)

# 数据质量检查
invalid = ['-', '暂无数据', 'N/A', '', None]
valid_cells = sum(
    str(v).strip() not in invalid 
    for row in df.values for v in row
)

if valid_cells == 0:
    raise ValueError("No valid data found!")

# 保存
df.to_excel('output.xlsx', index=False)

错误处理与重试

必须重试满 5 次才能报告错误。这是保证成功率的关键。 

#!/bin/bash
MAX_RETRIES=5
RETRY_COUNT=0

while [ $RETRY_COUNT -lt $MAX_RETRIES ]; do
    data=$(openclaw browser evaluate "/* extraction */")
    
    # 验证数据有效性
    if [ -n "$data" ] && 
       [[ "$data" != *"暂无数据"* ]] && 
       [[ "$data" != *"['-']"* ]]; then
        echo "✅ SUCCESS"
        break
    fi
    
    RETRY_COUNT=$((RETRY_COUNT + 1))
    echo "Attempt $RETRY_COUNT failed, retrying..."
    sleep 3
done

if [ $RETRY_COUNT -eq $MAX_RETRIES ]; then
    echo "❌ FAILURE after $MAX_RETRIES attempts"
    exit 1
fi

Skill 打包与安装

1. 使用 skill-creator 初始化

python3 scripts/init_skill.py spa-extractor \
    --path skills/public \
    --resources scripts,references

2. 编写 SKILL.md

关键：description 字段必须包含触发条件，因为 skill 是根据 description 来匹配用户请求的。

SKILL.md 头部示例

---
name: spa-extractor
description: |
    Extract data from JavaScript-heavy Single Page Applications (SPAs) 
    using OpenClaw browser automation. Use when dealing with Vue/React/Angular 
    apps that render content dynamically, require waiting for DOM hydration, 
    involve complex table extraction, or need data exported to Excel.
---

3. 打包与安装

# 打包
python3 scripts/package_skill.py spa-extractor ./dist

# 生成 spa-extractor.skill 文件（实际是 zip 格式）

总结与建议

写好 Skill 的关键

  1. 明确的触发描述 - description 要准确描述技能能力和使用场景
  2. 具体的工作流程 - 提供可执行的步骤，而不是泛泛而谈
  3. 丰富的代码示例 - 示例比解释更有效
  4. 实用的工具脚本 - 将复杂逻辑封装到 scripts/ 中
  5. 容错与重试机制 - 真实世界充满不确定性

常见框架模式参考

框架 表格行选择器 Loading 选择器 下拉列表位置
Element UI .el-table__row .el-loading-mask body 末尾
Ant Design .ant-table-row .ant-spin 相对定位

Skill 开发是一门平衡艺术：既要提供足够的指导让 AI 能够正确执行任务， 又要保持简洁避免上下文膨胀。希望这篇指南能帮助你开发出强大的 OpenClaw Skills！
